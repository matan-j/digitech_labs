# pedagogic-method.md — Digitech Teaching Method Rules

Apply these rules to ALL curriculum, lesson plan, module, and activity outputs.

---

## RULE 1 — Open with a Curiosity Trigger

Every unit, module, or lesson must start with ONE of:
- A strong provocative question
- A real-world scenario or case
- A surprising fact or counterintuitive result
- A challenge or problem to solve
- A short story (2–4 sentences)
- A relatable mistake or misconception to examine
- A visual prompt or image description

❌ Never open with: "Today we will learn about..."
✅ Open with: "A company lost $4 million because of one data error. How?"

---

## RULE 2 — Theory Blocks Are Short and Connected

- Maximum 15 continuous minutes of theory
- After every theory block → immediate activity
- Theory must be followed by: "Try this" / "Now you do" / "Discuss this"
- No walls of text in lesson plans or slides

---

## RULE 3 — Every Module Has an Active Element

Choose minimum one per module:

| Type           | Example                                      |
|----------------|----------------------------------------------|
| Activity       | Categorize this dataset                      |
| Exercise       | Fill in the worksheet section                |
| Discussion     | What would you do if this was your data?     |
| Build task     | Create a mini chart from this table          |
| Challenge      | Find the error in this dataset               |
| Reflection     | Write one thing you'd change                 |
| Game element   | Earn points by correctly tagging data types  |
| Group work     | Compare your answers with your table         |
| Decision task  | Choose which chart type fits this data       |

---

## RULE 4 — Tangible Learner Output is Mandatory

Every lesson or mini-unit must end with a student-created output:

| Output Type          | Example                                      |
|----------------------|----------------------------------------------|
| Completed worksheet  | Analyzed dataset with answers                |
| Mini presentation    | 2-slide summary of findings                  |
| Labeled infographic  | Student-created visual from data             |
| Written reflection   | 3 sentences about what they learned          |
| Solved challenge     | Corrected dataset or fixed query             |
| Digital artifact     | Google Sheet with student analysis           |
| Group solution       | Table poster or shared doc                   |

---

## RULE 5 — Real-World Relevance is Non-Negotiable

Every module must explicitly answer these three questions for the student:
1. **Why does this matter?** (motivation)
2. **Where do you actually see this?** (context)
3. **How could you use this?** (application)

These can be embedded in the opening trigger, a slide, or a discussion prompt — but they must appear.

---

## RULE 6 — Understanding Checks Are Inline

Do not wait until the end to assess comprehension.

Include at least one inline check per module:
- A quick question (verbal or written)
- A 2-option decision ("which is correct: A or B?")
- A thumbs up / thumbs down moment
- A short exit ticket
- A partner check ("explain it to the person next to you")

---

## RULE 7 — Sharing and Presentation Layer

When feasible, design a moment for students to:
- Show their output to a partner
- Present one finding to the class
- Compare results across tables
- Post to a shared board (physical or digital)

This is not optional in a full lesson — it is part of the learning structure.

---

## RULE 8 — Instructional Flow Per Module

Use this template when designing a module:

```
MODULE FLOW TEMPLATE
─────────────────────
[1] Opening Trigger          (2–3 min)
[2] Short Theory / Context   (5–12 min, max 15)
[3] Guided Example           (3–5 min)
[4] Student Activity         (8–15 min)
[5] Share / Check / Reflect  (3–5 min)
[6] Transition / Bridge      (1–2 min)
[7] Learner Output Check     (end of module or unit)
```

---

## RULE 9 — No Passive Modules

A module that contains only explanation, slides, and reading is incomplete.
Passive modules must be flagged in QA as: `⚠️ PASSIVE_MODULE — add activity`.

---

## RULE 10 — Interactive Asset in Every Full Course

Every complete course (4h+) must include at least one:
- Game
- Quiz with scoring
- Simulation or challenge flow
- Live competitive activity
- Interactive app or tool

This is not optional. It is a Digitech signature.
